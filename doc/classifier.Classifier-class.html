source code 

Initializes a classifier object with the specified settings.

Parameters:
  • feat_vectorizer (DictVectorizer) - A DictVectorizer that transforms lists of feature-value mappings to vectors.
  • scaler (Scaler) - A pre-fit scaler for the data that this classifier will be processing.
  • label_dict (dict) - Maps from class/label names to integers.
  • inverse_label_dict (list of basestring) - Maps from integers back to label strings.
  • model_type (basestring) - Type of estimator to create. Options are: 'logistic', 'svm_linear', 'svm_radial', 'naivebayes', 'dtree', 'rforest', and 'gradient'
  • probability (bool) - Should classifier return probabilities of all classes (instead of just class with highest probability)?
  • model_kwargs (dict) - A dictionary of keyword arguments to pass to the initializer for the specified model.

train(self, examples, clear_vocab=False, param_grid=None, grid_search_folds=5, grid_search=True, grid_objective=<function f1_score_micro at 0x42987d0>)

source code 

Train a classification model and return the model, score, feature vectorizer, scaler, label dictionary, and inverse label dictionary.

Parameters:
  • examples (array) - The examples to train the model on.
  • clear_vocab (bool) - Wipe out the feature vectorizer, scaler, label dictionary, and inverse label dictionary. This should be done if you're retraining a Classifier on a completely different data set (with different features).
  • param_grid (list of dicts mapping from basestrings to lists of parameter values) - The parameter grid to search through for grid search. If unspecified, a default parameter grid will be used.
  • grid_search_folds (int) - The number of folds to use when doing the grid search.
  • grid_search (bool) - Should we do grid search?
  • grid_objective (function) - The objective functino to use when doing the grid search.
Returns: float
The best grid search objective function score, or 0 if we're not doing grid search.

cross_validate(self, examples, stratified=True, clear_vocab=False, cv_folds=10, grid_search=False, grid_search_folds=5, grid_objective=<function f1_score_micro at 0x42987d0>, prediction_prefix=None, param_grid=None)

source code 

Cross-validates a given model on the training examples.

Parameters:
  • examples (array) - The data to cross-validate classifier performance on.
  • stratified (bool) - Should we stratify the folds to ensure an even distribution of classes for each fold?
  • clear_vocab (bool) - Wipe out the feature vectorizer, scaler, label dictionary, and inverse label dictionary. This should be done if you're retraining a Classifier on a completely different data set (with different features).
  • cv_folds (int) - The number of folds to use for cross-validation.
  • grid_search (bool) - Should we do grid search when training each fold? Note: This will make this take *much* longer.
  • grid_search_folds (int) - The number of folds to use when doing the grid search.
  • grid_objective (function) - The objective function to use when doing the grid search.
  • param_grid (list of dicts mapping from basestrings to lists of parameter values) - The parameter grid to search through for grid search. If unspecified, a default parameter grid will be used.
  • prediction_prefix (basestring) - If saving the predictions, this is the prefix that will be used for the filename. It will be followed by ".predictions"
Returns: list of 4-tuples
The confusion matrix, overall accuracy, per-class PRFs, and model parameters for each fold.
